RDF graph terms are represented by the type. These are RDF terms: URI, literal or blank node. In the graph, the predicate of a triple can only be a URI.

All strings are considered to be UTF8-encoded. The module provides some functions to manipulate UTF8 strings.

The module defines functions to create and print RDF terms. For example, creates a literal RDF term from an integer, setting the URI of the datatype to <xsd:integer>:

let int_lit = Rdf_term.mk_literal_int 42;; let term42 = Rdf_term.Literal int_lit;;

Let's get a string representation of the term:

print_endline (Rdf_term.string_of_term term42) ;;

The module defines the type to represent URIs and contains functions to create URIs and perform other manipulations:

let uri_s = "http://foo.bar.net/";; let uri = Rdf_iri.iri uri_s;; let uri2 = Rdf_uri.set_fragment uri "id42";; Rdf_uri.compare uri uri2;; Rdf_iri.string uri2;;

The module defines some commonly used URIs: datatypes, classes, property names, ...:

Rdf_iri.string (Rdf_rdf.xsd_integer);; Rdf_iri.string (Rdf_rdf.xsd_double);; Rdf_iri.string (Rdf_rdf.rdf_about);;
