RDF graph nodes are represented by the type. These nodes are RDF terms: URI, literal or blank node. In the graph, the predicate of a triple is also a node, even though by now the RDF specification only authorize URIs.

The module defines functions to create and print nodes. For example, creates a literal RDF term from an integer, setting the URI of the datatype to <xsd:integer>:

let int_lit = Rdf_node.mk_literal_int 42;; let node42 = Rdf_node.Literal int_lit;;

Let's get a string representation of the node:

print_endline (Rdf_node.string_of_node node42) ;;

The module defines the type to represent URIs and contains functions to create URIs and perform other manipulations:

let uri_s = "http://foo.bar.net/";; let uri = Rdf_uri.uri uri_s;; let uri2 = Rdf_uri.set_fragment uri "id42";; Rdf_uri.compare uri uri2;; Rdf_uri.string uri2;;

The module defines some commonly used URIs: datatypes, classes, property names, ...:

Rdf_uri.string (Rdf_rdf.xsd_integer);; Rdf_uri.string (Rdf_rdf.xsd_double);; Rdf_uri.string (Rdf_rdf.rdf_about);;
