The Sparql 1.1 query language is implemented, with some exceptions:

  • The following built-in calls are still missing: UUID, UCASE, LCASE, langMatches (only based on prefix by now, must implement language matching according to RFC 4647), TIMEZONE, TZ, SHA384, SHA512.
  • Subqueries are not handled yet.
  • DESCRIBE queries are not handled yet.
  • Check of variable scope is not implemented yet.

The parsing of Turtle and Sparql queries is based on Ulex and Menhir. But Ulex does not offer convenient way to handle positions. By now, some hack allows to get the correct position for syntax errors, but evaluation errors contains incorrect positions: line numbers are correct, but character indices are offset from the beginning of the file or string which was parsed.

  • A field "context" or "data" could be added to triples in the graphs, as it is in Redland RDF libraries.
  • Implement Sparql 1.1 Update.
