Module Rdf_4s_lwt

module Rdf_4s_lwt: sig .. end
Rdf 4Store - This Module implement the http binding of 4Store

Bindings
In this document: uri is the server's uri (including the port) query is the SPARQL query graph_uri is the uri of the graph where the query is going to be executed. graph_uri if is equal to uri it is executed on the default graph data_type are set at "x-turtle" by default.
val get : ?graph:Rdf_graph.graph ->
       base:Rdf_iri.iri ->
       ?accept:string ->
       Rdf_uri.uri ->
       Rdf_sparql_protocol.in_message -> Rdf_sparql_protocol.out_message Lwt.t
get uri ?default_graph_uri ?named_graph_uri query query If you could like to select a named-graph, it has to be made in the query. This method allows: select/ask/describe query.
val post_update : Rdf_uri.uri -> string -> Rdf_sparql_protocol.out_message Lwt.t
post_update uri query query If you would like to update a named-graph, it has to be made in the query. This method allows: update/insert/delete query.
val delete : Rdf_uri.uri -> Rdf_uri.uri -> Rdf_sparql_protocol.out_message Lwt.t
delete uri graph_uri This method removes the entire graph_uri.
val put : Rdf_uri.uri ->
       string ->
       ?data_type:string -> Rdf_uri.uri -> Rdf_sparql_protocol.out_message Lwt.t
put uri data ?data_type graph_uri This method allows to replace initial data from graph_uri by data. data is allowed as rdf-xml type.
val post_append : Rdf_uri.uri ->
       string ->
       ?data_type:string -> Rdf_uri.uri -> Rdf_sparql_protocol.out_message Lwt.t
post_append uri data ?data_type graph_uri This method allow to append data into graph_uri. data is allow as ttl type.
