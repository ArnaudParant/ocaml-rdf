RDF terms and URIs

RDF graph nodes are represented by the Rdf_node.node type. These nodes are RDF terms: URI, literal or blank node. In the graph, the predicate of a triple is also a node, even though by now the RDF specification only authorize URIs.

All strings are considered to be UTF8-encoded. The Rdf_utf8 module provides some functions to manipulate UTF8 strings.

The Rdf_node module defines functions to create and print nodes. For example, Rdf_node.mk_literal_int creates a literal RDF term from an integer, setting the URI of the datatype to <xsd:integer>: 

# let int_lit = Rdf_node.mk_literal_int 42;;
val int_lit : Rdf_node.literal =
  {Rdf_node.lit_value = "42"; Rdf_node.lit_language = None;
   Rdf_node.lit_type = Some <abstr>}
# let node42 = Rdf_node.Literal int_lit;;
val node42 : Rdf_node.node =
  Rdf_node.Literal
   {Rdf_node.lit_value = "42"; Rdf_node.lit_language = None;
    Rdf_node.lit_type = Some <abstr>}

Let's get a string representation of the node: 

# print_endline (Rdf_node.string_of_node node42) ;;
"42"^^<http://www.w3.org/2001/XMLSchema#integer>
- : unit = ()

The Rdf_uri module defines the type Rdf_uri.uri to represent URIs and contains functions to create URIs and perform other manipulations: 

# let uri_s = "http://foo.bar.net/";;
val uri_s : string = "http://foo.bar.net/"
# let uri = Rdf_uri.uri uri_s;;
val uri : Rdf_uri.uri = <abstr>
# let uri2 = Rdf_uri.set_fragment uri "id42";;
val uri2 : Rdf_uri.uri = <abstr>
# Rdf_uri.compare uri uri2;;
- : int = -1
# Rdf_uri.string uri2;;
- : string = "http://foo.bar.net/#id42"

The Rdf_rdf module defines some commonly used URIs: datatypes, classes, property names, ...: 

# Rdf_uri.string (Rdf_rdf.xsd_integer);;
- : string = "http://www.w3.org/2001/XMLSchema#integer"
# Rdf_uri.string (Rdf_rdf.xsd_double);;
- : string = "http://www.w3.org/2001/XMLSchema#double"
# Rdf_uri.string (Rdf_rdf.rdf_about);;
- : string = "http://www.w3.org/1999/02/22-rdf-syntax-ns#about"
